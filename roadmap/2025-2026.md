## Phase 0: Foundation (2025 Q4) - TRL 3

### Objective
Establish mathematical foundations and reference implementations
for both the HSL authentication layer and post-quantum analysis
framework.

### Deliverables

| ID | Deliverable | Status | Reference |
|----|-------------|--------|-----------|
| F0-01 | HSL phase-based challenge-response module | Done | Paper 4, `hsl/hsl_module.py` |

| F0-06 | Consistency projection P_C (7/64 profiles) | Done | Paper 2, `post-quantum/quantum_profiles.py` |
| F0-07 | Security architecture document | Done | `docs/security-architecture.md` |

## Phase 1: Laboratory Validation (2026 Q1-Q2) - TRL 3 to 4

### Objective
Validate HSL and post-quantum modules in a controlled laboratory
environment with systematic benchmarking against established standards.

### Deliverables

| ID | Deliverable | Target | Reference |
|----|-------------|--------|-----------|
| F1-01 | HSL handshake benchmark vs TLS 1.3 | Q1 2026 | Paper 4 |
| F1-02 | Intrusion detection ROC curves | Q1 2026 | Paper 4 |
| F1-03 | LFSR period verification (2^32 - 1) | Q1 2026 | Paper 4 |
| F1-04 | rho_3 bound empirical validation | Q1 2026 | Paper 2 |
| F1-05 | Consistency projection accuracy tests | Q2 2026 | Paper 2 |
| F1-06 | ML-KEM-768 integration prototype | Q2 2026 | NIST FIPS 203 |
| F1-07 | IEEE/ACM conference paper submission | Q2 2026 | Both papers |

### Key Metrics

| Metric | Current Target | Success Criteria |
|--------|---------------|-----------------|
| HSL handshake size | ~200 bytes | < 250 bytes |
| HSL handshake latency | ~0.3 ms P99 | < 1 ms P99 |
| Intrusion detection TPR | > 95% | > 90% at 1% FPR |
| rho_3 bound violations | 0% | 0 violations |
| Consistency projection | P_C >= 0.95 for consistent states | Verified |
